¿Qué es Electron?

Un framework para crear aplicaciones de escritorio nativas con tecnologías web como JavaScript, HTML y CSS.

¿Qué necesito saber?

  • Node.js (JavaScript)
  • HTML/CSS

¿Qué es Node.js?

  • Un entorno de ejecución multiplataforma para JavaScript construido con el motor de JavaScript V8 de Chrome
  • Es código abierto
  • Orientado a servidor (pero no solo usado en él)
  • Es asíncrono y con un modelo de operaciones de E/S sin bloqueo y orientado a eventos

¿Cómo funciona Electron?

  • Usando Node.js y un subconjunto mínimo de librerías de Chromium
  • La interfaz de la aplicación es simplemente HTML/CSS
  • Una vez realizada la aplicación, se genera el ejecutable para cualquier plataforma que queramos

¿Cómo funciona Electron? (internamente)

  • Tenemos un proceso principal, que es nuestra aplicación
  • También tenemos un proceso renderizador, que usa la arquitectura multiproceso de Chromium
  • Cada página web es un proceso independiente
  • El proceso principal controla todas las páginas web y sus procesos de renderizado
  • Ventaja: las páginas web no tienen acceso a recursos nativos, sin embargo, con Electron podemos usar el API de Node.js, lo que nos permite interactuar a un nivel más bajo.

¿Cómo funciona Electron? (funcionalmente)

  • En la aplicación principal se crean instancias BrowserWindow
  • Esa instancia es nuestra ventana de la interfaz, nuestro proceso renderizar que carga una página
  • Programación orientada a eventos:
    • electron.app.on('ready', () => { ... });
    • electron.BrowserWindow.on('closed', onClosed);

¿Cómo lo pongo en marcha?

  • Ejecutar: electron .
  • Construir: electron-packager . --out=dist --asar --linux64
